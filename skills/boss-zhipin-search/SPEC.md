# BOSS直聘信息检索工具 - 结构化需求文档

## 1. 项目概述

**项目名称**: BOSS直聘岗位搜索助手 (boss-zhipin-search)

**项目类型**: OpenClaw Skill Tool

**核心功能**: 帮助用户快速检索BOSS直聘平台上的岗位信息，支持多维度筛选，并生成结构化的岗位汇总报告。

**目标用户**: 求职者、HR、猎头

---

## 2. 功能需求

### 2.1 核心功能列表

| 序号 | 功能名称 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| F1 | 联网搜索 | 通过BOSS直聘搜索接口获取岗位数据 | P0 |

## 3. 技术方案

### 3.1 搜索URL格式

```
https://www.zhipin.com/web/geek/job?query={关键词}&city={城市代码}&salary={薪资代码}&experience={经验代码}&degree={学历代码}&page={页码}
```

### 3.2 数据字段

从BOSS直聘获取的岗位数据应包含以下字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| jobId | 岗位ID | - |
| jobName | 岗位名称 | Python开发工程师 |
| brandName | 公司名称 | 字节跳动 |
| salaryDesc | 薪资范围 | 15-30K |
| cityName | 城市 | 北京 |
| areaName | 区域 | 朝阳区 |
| jobExperience | 经验要求 | 3-5年 |
| jobDegree | 学历要求 | 本科 |
| brandScaleName | 公司规模 | 10000人以上 |
| brandIndustry | 所属行业 | 互联网 |
| welfareList | 福利标签 | [五险一金, 弹性工作, 股票期权] |
| skills | 技能标签 | [Python, Django, MySQL] |
| jobDetailUrl | 岗位详情链接 | - |
| updateTime | 发布时间 | 3天前 |

### 3.3 技术实现方案

#### 方案A: 浏览器自动化 (推荐)
- 使用 `browser` 工具控制浏览器访问BOSS直聘搜索页面
- 优点: 无需Cookie，可以看到完整的渲染页面
- 缺点: 需要处理反爬虫检测

#### 方案B: Web Search API
- 使用 `web_search` 工具搜索BOSS直聘岗位
- 优点: 简单快速
- 缺点: 搜索结果数量有限

#### 方案C: API接口 (长期方案)
- 逆向BOSS直聘API接口
- 优点: 效率高，可控性强
- 缺点: 难度大，可能失效

### 3.4 反爬虫应对策略

1. **User-Agent轮换**: 每次请求使用不同的User-Agent
2. **请求间隔**: 每次请求间隔1-2秒
3. **Cookie处理**: 定期更新Cookie
4. **IP轮换**: 使用代理IP池 (可选)

## 6. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 搜索结果<20条 | 返回全部结果，提示用户扩大搜索范围 |
| 搜索结果为0 | 提示用户调整筛选条件 |
| 反爬虫拦截 | 切换搜索策略或提示用户稍后重试 |
| 网络超时 | 重试3次后返回错误信息 |
| 城市名称不识别 | 提示用户选择预设城市列表 |
